# UX-COPY-01 - Raccourcissement des textes UI

## Diagnostic global

Les derniers lots UX avaient déjà réduit la densité visuelle, mais plusieurs
pages spécialisées conservaient des phrases longues directement dans
l'interface. Le risque principal était une lecture lente sur mobile, surtout en
usage terrain.

Le lot raccourcit les textes visibles sans retirer les garde-fous.

## Spécificités desktop vs mobile

Sur mobile, les textes deviennent plus scannables: titres courts, détails de
cartes réduits et limites formulées en phrases simples.

Sur desktop, les messages restent compréhensibles, mais les explications longues
doivent vivre dans la documentation ou les revues UX plutôt que dans chaque
surface applicative.

## Plan d'action appliqué

- Raccourcir les descriptions des pages visites, tâches, matériel et
  transhumance.
- Raccourcir les garde-fous de fin de page.
- Raccourcir quelques textes admin, catalogue modules et onboarding.
- Conserver les mentions de développement, absence d'authentification réelle,
  absence d'IA, absence de GPS actif et absence d'actions dangereuses.

## Règles sources

- Bastien & Scapin: réduire la charge de travail.
- Loi de Miller: garder les messages courts et groupés.
- Loi de Hick: éviter de multiplier les signaux explicatifs.
- WCAG 2.2: améliorer la lisibilité sans retirer l'information de statut.

## Limites

Ce lot ne modifie aucun flux, aucune action serveur, aucune donnée et aucune
permission. Il ne remplace pas une future revue éditoriale complète.
